Сделайте:
1. Проанализируйте `agent_sessions`, `flow_events`, комментарии Issue/PR.
2. Сформируйте диагностику причин повторяющихся сбоев/замечаний.
3. Подготовьте change-set с трассировкой источников.

Артефакты результата:
- self-improve diagnosis;
- proposed change-set (docs/prompts/instructions/tooling);
- rationale + evidence links.

Критерий завершения:
- подготовлен воспроизводимый пакет улучшений под PR.
