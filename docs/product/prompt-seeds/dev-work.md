# Prompt Seed: Dev Work

## Назначение
Базовый seed-шаблон для выполнения задач разработки (`run:dev`).
Это не финальный effective prompt рантайма, а fallback-структура для bootstrap (`repo seed`), которая затем может быть переопределена в БД.

## Обязательная структура
1. Цель задачи и ожидаемый результат.
2. Ограничения архитектуры и source of truth документы.
3. Пошаговый план выполнения.
4. Реализация + тестирование + обновление документации.
5. Отчёт: что сделано, как проверено, риски/ограничения.

## Обязательные требования к ответу агента
- Не нарушать `docs/design-guidelines/**`.
- Поддерживать typed contracts (без `any` в transport-контрактах).
- При изменении поведения обновлять проектную документацию синхронно с кодом.
- Любые неопределённости фиксировать явно и запрашивать решение только по блокерам.
- Перед завершением цикла учитывать pre-review контур: отдельный `reviewer` агент проверяет задачу/доки/гайды и оставляет inline комментарии в PR.

## Локали и рендер
- Seed поддерживается минимум в `ru` и `en` (через записи в БД при инициализации платформы).
- Язык effective шаблона выбирается по цепочке: `project locale -> system default locale -> en`.
- Финальный prompt рендерится с runtime-контекстом (namespace/slot/env/MCP/project/run).
